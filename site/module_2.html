Anatomy of a CSS Rule

Example of a CSS Rule:

p {
color: blue;
}
  1. The first line contains the selector. The rules given below the selector apply to every item within the HTML file with that tag. In this instance, every item with the paragraph tag.
  2. The second line contains the CSS declaration consisting of 2 parts.
    1. Property: properties are predefined by CSS. In this case, the property is color.
    2. Value: For every property there is a list of predefined values. In this case the value is blue.

Each CSS rule can contain zero or more declarations.

Example:

p {
color: blue;
font-size: 20px;
width: 200px;
}
The collection of all of these CSS rules is called a style sheet. Style sheets go in the head section of the document for now in the <style> tag. Every browser comes with default styles for each tag type.

Element, Class, and ID Selectors

Selectors tell our CSS rules what to apply to

Element Selector: specifies the element tag that the CSS rule will apply to. In the example above, the element selector is p.

Class Selector: Specifies a class that can be applied to elements selectively.

Example:

.blue {
color: blue;
}

Class selectors can be applied to elements individually with the format:

<p class="blue"> </p>
The text in this element will be blue.

ID Selector: Specifies an ID value that the CSS rule will apply to.

Example:

#name {
color: blue;
}

This will be applied to all elements with the ID "name":

<div id="name"> </div>
The text in this element will be blue.

Grouping Selectors: Multiple selectors can be grouped together, separating them with a comma. Both selectors will work to format content.

Example:

div, #name {
color: blue;
}

All elements type div or with id "name" will be blue.

Combining Selectors

Element with Class selector:
p.big {

This rule will apply to Every p element with class="big". There can't be any spaces between the two selectors.

Child Selector:
article > p {

This rule will target every p element that is a direct child of an article element.

Descendant Selector:
article p {

This rule will target any p element that is inside an article element at any level. Does not need to be a direct descendent. All of these combined selectors are not limited to elements and can be used with classes as well.

Adjacent Sibling Selector:
img + p {

This rule only applies to elements type p that are directly following elements type img, and both elements are a part of the same parent.

General Sibling Selector:
img ~ p {

This rule applies to all elements type p that come after an img element as long as they are a part of the same parent. The p element does not need to be directly after the img element.

Pseudo-Class Selectors

Pseudo-Class Selectors target only structures that can be targeted with simple combinations of regular selectors or structures that we want to style based on user-interactivity. For example, changing the style of a structure if the user hovers their mouse over it.

selector:pseudo-class {

selector is one of the previously discussed selectors. Pseudo-Classes include, but are not limited to:

:link - Applies to unvisited links. Usually used with a tags.
:visited - Applies to visited links. Usually applies to a tags.
:hover - Applies to tags only when the user hovers over the element.
:active - Applies to tags only while the user is actively clicking the element.
:nth-child() - Applies to the specified child. ex: section div:nth-child(4) { Specifies the 4th div child of a section element.

Style Placement

Style placement determines how reusable styles are and what styles override other styles. You can specify a style in multiple places:

  1. Inside the <style> tag in the head
  2. (Inline styling) Inside the element tag using style=""
    • You can include as many rules as you want, separating them with semicolons;
    • Least reusable way to style, thus least preferred.
  3. External style sheets
    • Added to the HTML file using the following syntax:
    • <link rel="stylesheet" href="file-name.css">
    • You almost always want all of your styles to be external from the HTML file itself.
The lower the rule is in the file, the higher precedence it has. Inline styling > head styles > external styles. Further, a style lower in the external styles or head styles takes precedence over styles above them.

Conflict Resolution

Conflict resolution helps us determine which styles will be used and how to resolve conflicting styles.

Key concepts to CSS

Origin - origin precedence kicks in when 2 or more rules are in conflict. Origin precedence specifies that the last declaration wins. When declarations target the same element, but they do not conflict, i.e. they don't format the same things, the declarations merge.
Merge - 2 or more declarations target the same element, but they do not conflict. The styles of both declarations are applied to the target element.
Inheritance - Every element within an element that has a rule targeted at it inherits that rule. If a rule is declared on the body tag, every element within the body tag inherits that rule, unless it is overridden.
Specificity - Most specific selector combination wins. We can determine which selector is most specific by keeping score, and determining which has the highest score. To find the scores, we order the selectors from left to right from most specific to least specific.
  1. inline style attribute (style="")
    • If there is an inline style attribute, this is scored as a 1, and everything else gets a 0. Giving us a score of 1000
  2. ID selectors
  3. Class, Pseudo-Class, Attribute Selectors
  4. Number of elements used in selector combination

Examples of specificity scores.

Example 1:

div p { color: green; }
This style selector is not inline, does not have an id, does not have a class, and has 2 elements.
Score: 0002

Example 2:

div #myParag { color: blue; }
This style selector is not inline, has 1 id, has 0 classes, and has 1 element.
Score: 0101

Example 3:

div.big p { color: green; }
This style selector is not inline, has 0 ids, has 1 class, has 2 elements.
Score: 0012

We can override any styles with more specificity by adding !important before the semicolon.

Example:

p { color: blue !important; }

This will become a maintenance nightmare. Avoid using unless you absolutely have to.

Styling Text

This section will display a few ways that we can style text using CSS.

  • font-family:
    • You sholud specify more than one font family because you are relying on the user to have these fonts installed on their computer.
    • A list of fonts and combinations can be found at This website
    • You also want to specify at the end "serif" or "sans-serif" in case none of the other fonts are installed.
  • color:
    • You can use color names: red, yellow, etc.
    • Hexidecimal values are better for precise colors (#0000ff)
  • font-style: Specifies whether you want something to be "italic" or "normal
  • font-weight:
    • Can be sepecified as "bold"
    • Or can be specified from 100 - 1000 in 100 point increments.
  • font-size: Specified in pixels "24px"
  • text-transform:
    • capitalize - capitalizes the first letter of every word.
    • lowercase - makes every letter lowercase
    • uppercase - makes every letter uppercase
  • text-align: aligns text within its block-level
    • center
    • justify
    • left
    • right
    • end

Relative Font Sizes.

px or pixels is an absolute font size. It will be the same on every device or screen. There are two types of relative font sizes.

  1. percents
    • percents change the font size based on whatever the default font sizes are.
    • font-size: 120%; would make the font 120% the size of the default font setting or the current font setting.
  2. em's
    • em's change the font size relative to what the current font size is.
    • if the current font size is 16px, and we use the style font-size: 2em; The font will be 32px.

The Box Model

In HTML, every element is considered a box. Each box consists of:

  1. content
  2. padding - Surronds each element in the body.
  3. border - Surrounds the padding.
  4. margin - Surrounds the entire body of our code. Also surrounds the border and padding of each element.
This is my content, surrounded by padding in blue, and border in black.
Going from smallest to largest. The size of each element is the sum of the size of the content box, padding box, and border box.

The width: style only changes the width of the content box, not the total width.
We can change this to include the padding and border by using the style box-sizing: border-box; in the rule.
Box-sizing is a style that is not inherited. We need to use the star(*) selector to apply box-sizing everywhere. The star selector applies styles to every element in the program.
Example: * { box-sizing: border-box; }

Margins: Margins for elements next to each other horizontally are cumulative. If 1 box has a margin of 25px and is horizontally next to another box with margin 30px. The total margin between them is 55px.

Margins next to each other vertically are not cumulative and collapse to the size of the larger margin. If a box with 25px margin is above a box with a 30px margin, the margin between the two boxes is 30px.

The Background Property

There are a lot of different things we can do to the backgrounds of content:

  • background-color: Sets the color of the background.
  • background-image: url("file-name.jpg") - sets image to background. Default repeats image to fill background size. Image always shows on top of background color.
  • background-repeat: - sets how the image is inserted as the bacground.
    • repeat - default
    • repeat-y - repeats the image vertically only
    • repeart-x - repeats the image horizontally only
    • no-repeat - only includes one image and may not fill the entire background
  • background-position: top right; - positions the image in the top right of the background. If you specify only one, the other defaults to center.
  • background: Allows us to combine all of the above together. This property overrides all other individually specified background properties. No specific order.

Most user interfaces today are made by floating elements.

When you float elements, the browser takes them out of the regular document flow. Margins of floated elements never collapse, they always accumulate.

float: right; will float the current property to the right.

In order to get the document back to regular document flow, we need to use the clear: property.

clear: left; will make sure nothing is floating to the left of an element.
clear: both; will make sure there are no floating elements to the left or the right.

Floating elements allow for very flexible site layouts.

Relative and Absolute Element Positioning

Static Positioning - Normal document flow. Defalut for all elements except html. Positioning offsets are ignored.

Relative Positioning - Element is positioned relative to its position in normal document flow. Offsets will position the element relative to where it would normally be. The element is not taken out of normal document flow. Even if it is moved, its orginal spot is reserved.

Positioning properties are: top:, bottom:, left:, right:. These properties move the object from the stated direction, so top: 50px; moves the object down 50px from the top edge.You can also use negative values

Absolute Positioning - All offsets are relative to the position of the nearest ancestor which has positioning set other than static. By default, html is the only element that has non-static positioning set on it. Element is taken out of normal document flow if positioning is set to absolute.

Use position: absolute; to make positioning absolute. We then use the offset properties (bottom, top, left, right) to move the element.
Note - bottom: 10px; moves the absolute element up 10 px from the bottom of the other element.

Media Queries

Media queries let you display certain styles based on the target device (height, width, orientation).

Media Query Syntax:
@media (max-width: 767px) {
p { color: blue; }
}

We put all of the styles we want to display on that type of device inside the media query. You can combine media operators by using logical operators, such as and Commas are equivalent to or logical operators.

Types of media query properties:
min-width, max-width, min-height, max-height

Responsive Design

Responive design was born mainly due to the explosion of mobile devices. There are so many devices, but you cannot ignore any of them.

A responsive website is a site designed to adapt its layout to the viewing environment by using fluid, proportion-based grids, flexible images, and CSS3 media queries.

The most important part of responsive design is the layout. The most common layout is the 12-column grid responsive layout.

12-column grid responsive layout - We evenly split content across 12 evenly sized grids. 1 column = 8.33% of all 12 columns. We can also nest grids inside columns.

<meta name="viewport" content="width=device-width, initial-scale=1">
The meta tag above informs the browser that the website is responsive and that it doesn't need to zoom out.
width=device-width tells the browser to not try to zoom out. initial-scale=1 sets the initial scale to 1.

The Bootstrap Grid System

This is one of the most central components of Bootstrap.
The grid system is built off of the following lines.

<div class="container"> Bootstrap grid always has to be inside a container.
<div class="row"> Creates horizontal groups of columns.
<div class="col-md-4">Col 1</div> Defines the size of columns within the row.
...
</div>
</div>

The above code does not need to be placed in div elements, meaning you can use semantic elements such as header and nav.
container class - Has predetermined, fixed width that is still responsive based on the width of the browser. In other words, it has a certain width at one break point, and a different width at another break point.
container-fluid class - alternative to container class. This class stretches content the whole length of the browser and provides consistant padding around your grid and other content.
row class- creates horizontal groups of columns. The columns collapse and interact with each other as a group but independently from columns in another row. This class applies a negative margin to counteract the margin of the container. If there was no negative margin, the content in the columns would not line up with the content outside the rows.
Column classes - Define the size of columns in a row.

Column class format: col-SIZE-SPAN

Size: Screen width range identifier (md=medium, lg=large(width=1200px+), etc.) Sizes can be found in documentation. Columns will collapse (stack) below the given size, unless another rule applies.
Span: How many columns the element should span (1-12).
