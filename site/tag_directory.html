A

<a>
Inline
with the href attribute, creates a hyperlink to web pages, files, email addresses, locations in the same page, or anything else a URL can address.
<abbr>
Inline
Represents an abbrviation or acronym; the optional title attribute can provide an expansion or description for the abbreviation.
<address>
Indicates that the enclosed content provides contact information for a person or people, or for an organization.
Content Categories: Flow content, palpable content
Permitted Content: Flow content, but no <address> elements, no heading content (h1-h6, hgroup), no header or footer element.
<article>
Represents a self-contained composition in a document, page, application, or site, which is intended to be independently distributable or reusable.
<aside>
Represents a portion of a document whose content is only indirectly related to the document's main content.

B

<b>
Inline
Used to draw reader's attention to the element's contents, which are not otherwise granted special importance.
<bdi>
Inline
Bidirectional Isolate element
Tells the browser's bidirectional algorithm to treat the text it contains in isolation from its surrounding text.
This is particularly useful when a website dynamically inserts some text and doesn't know the directionality of the text being inserted.
<bdo>
Inline
Bidirectional Text Override element
Overrides the current directionality of text, so that the text within is rendered in a different direction.
<blockquote>
HTML Block Quotation Element
Indicates that the enclosed text is an extended quotation. Usually, this is rendered visually by indentation. A URL for the source of the quotation may be given using the cite attribute, while a text representation of the source can be given using the cite element.
<br>
Inline
Produces a line break in text (carriage-return). It is useful for writing a poem or an address, where the division of lines is significant.

C

<cite>
Inline
Is used to describe a reference to a cited creative work, and must include the title of that work.
<code>
Inline
Displays its contents styled in a fashion intended to indicate that the text is a short fragment of computer code.

D

<data>
Inline
Links a given piece of content with a machine-readable translation. If the content is time- or date-related, the time element must be used.
<dd>
Provides the description, definition, or value for the preceding term (dt) in a description list (dl).
<dfn>
Inline
Definition element
Used to indicate the term being defined within the context of a definition phrase or sentence.
<div>
Content Division Element
The generic container for flow content.
Content Categories: Flow content, palpable content
Permitted Parents: Any element that accepts flow content.
Note: div should only be used when no other semantic element is appropriate.
<dl>
Element represents a description list. The element encloses a list of groups of terms (specified using the dt element) and descriptions (provided by the dd elements). Common uses for this element are to implement a glossary or to display metadata (a list of key-value pairs).
<dt>
Specifies a term in a description or definition list, and as such must be used inside a dl element.

E

<em>
Inline
Marks that text has stress emphasis. The em element can be nested, with each level of nesting indicating a greater degree of emphasis.

F

<figcaption>
Figure Caption Element
Represents a caption or legend describing the rest of the contents of its parent figure element.
<figure>
Represents self-contained content, potentially with an optional caption, which is specified using the figcaption element.
<footer>
Represents a footer for its nearest sectioning content or sectioning root element. A footer typically contains information about the author of a section, copyright data or links to related documents.

H

<header>
Represents introductory content, typically a group of introductory or navigational aids. It may contais someheading elements but also a logo, a search form, an author name, and other elements.
<hr>
Empty element
Doesn't need an end tag
Represents a thematic break between paragraph-level elements: for example, a change of scene in a story, or a shift of topic within a section.
<h1> <h2> <h3> <h4> <h5> <h6>
Represents 6 levels of section headings, h1 is the highest, h6 is the lowest.
<hgroup>
Represents a multi-level heading for a section of a document. It groups a set of h1-h6 elements.

I

<i>
Inline
Idiomatic Text element
Represents a range of text that is set off from the normal text for some reason, such as idiomatic text, technical terms, taxonomical designations, among others.

K

<kbd>
Inline
Keyboard Input element
Represents a span of inline text denoting textual user input from a keyboard, voice input, or any other text entry device.

L

<li>
Used to represent an item in a list.

M

<main>
Represents the dominant content of the body of a document. The main content area consists of content that is directly related to or expands upon the central topic of a document, or the central functionality of an application.
<mark>
Inline
Represents text which is marked or highlighted for reference or notation purposes, due to the marked passage's relevance or importance in the enclosing text.

N

<nav>
Represents a section of a page whose purpose is to provide navigation links, either within the current document or to other documents. Common examples of navigation sections are menus, tables of contents, and indexes.

O

<ol>
Represents an ordered list of items. Typically rendered as a numbered list.

P

<p>
Represents a pragraph.
Block-level element that will close if another block-level element is parsed before the closing tag.
<pre>
Represents preformatted text which is to be presented exactly as written in the HTML file.

Q

<q>
Inline
Indicates that the enclosed text is a short inline quotation. Most modern browsers implement this by surround the text in quotation marks.

R

<rb>
Inline
Ruby Base element
Used to delimit the base text component of a ruby annotation, i.e. the text that is being annotated.
<rp>
Inline
Ruby Fallback Parenthesis element
Used to provide fall-back parenthesis for browsers that do not support display of ruby annotations using the ruby element.
<rt>
Inline
Ruby Text element
Specifies the ruby text component of a ruby annotation, which is used to provide pronunciation, translation, or transliteration informtation for East Asian typography. The rt element must always be contained within a ruby element.
<rtc>
Inline
Ruby Text Container element
Embraces semantic annotations of characters presented in a ruby of rb elements used inside of ruby element. rb elements can have both pronunciation (rt) and semantic (rtc) annotations.
<ruby>
Inline
Represents small annotations that are rendered above, below, or next to base text, usually used for showing the pronunciation of East Asian characters. It can also be use for annotating other kinds of text, but this usage is less common.

S

<s>
Inline
Renders text with a strikethrough, or a line through it. Use the s element to represent things that are no longer relevant or no longer accurate. However, the s element is not appropriate when idicating document edits; for that, use the del and ins elements.
<samp>
Inline
Used to enclose inline text which represents sample (or quoted) output from a computer.
<section>
Represents a standalone section within the HTML document. Used when there isn't a more specific semantic element to represent it.
<small>
Inline
Represents side-comments and small print, like copyright and legal text, independent of its styled presentation. By default, it renders text within it one-font-size smaller.
<span>
Inline
Generic inline container for phrasing content, which does not inherently represent anything. It can be used to group elements for styling purposes, or because they share attribute values, such as lang.
<strong>
Inline
Strong Importance element
Indicates that its contents have strong importance, seriousness, urgency. Browsers typically render the contents in bold type.
<sub>
Inline
Subscript element
Specifies inline text which should be displayed as subscript for solely typographical reasons.
<sup>
Inline
Superscript element
Specifies inline text which should be displayed as superscript for solely typographical reasons.

T

<time>
Inline
Represents a specific time period.

U

<u>
Inline
Unarticulated Annotation element
Represents a span of inline text which should be rendered in a way that indicates that it has a non-textual annotation.
<ul>
Represents an unordered list of items. Typically rendered as a bulleted list.

V

<var>
Inline
Represents the name of a variable in a mathematical expression or a programming context.

W

<wbr>
Inline
Represents a word break opportunity, a position within text where the browser may optionally break a line, though its line-breaking rules would not otherwise create a break at that location.
